Суть задания: создать коллекцию в Postman с запросами для полного тестирования метода, описание которого дано ниже.
URL в методе должен быть указан как глобальная переменная (кандидат не имеет доступа к ресурсу).
Токены авторизации в методе должен быть указаны как глобальные переменные (кандидат не имеет доступа к данным авторизации).
Запросы должны быть составлены таким образом, чтобы проверяющий мог их запустить, указав только URL и Токен в глобальных переменных.
В случае, если возникает вопрос, который не покрывает Документация, которая описана ниже,
кандидат должен сам выбрать любое решение, не противоречащее постановке.

Описание системы: система управления проектами предприятия
Метод POST позволяет создать новый проект в системе
Адрес: /api/projects
Авторизация: требуется
POST /api/projects
